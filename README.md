# SW-Project

## 프로젝트 소개

본 프로젝트는 오픈소스SW개론 최종 프로젝트로 진행하는 교내 중고거래 플랫폼이다.

교내 학생들이 사용하지 않는 물건을 판매글로 등록하고, 다른 학생들이 등록된 물품을 조회할 수 있도록 하는 웹 서비스를 목표로 한다.

본 프로젝트는 HTML, CSS, JavaScript를 사용하여 구현하며, GitHub를 활용하여 팀원별 기능 분담, 브랜치 관리, Pull Request 검토, Issue 관리를 진행한다.

## 프로젝트 주제

교내 중고거래 플랫폼

## 주제 선정 이유

대학생들은 전공책, 기숙사 물품, 전자기기, 생활용품 등을 자주 사고팔지만, 기존 중고거래 플랫폼은 지역 중심으로 운영되어 교내 학생 간 거래에 특화되어 있지 않다.

따라서 본 프로젝트에서는 같은 학교 학생들이 물품을 쉽게 등록하고 조회할 수 있는 간단한 교내 중고거래 플랫폼을 구현하고자 한다.

## 브랜치 전략

본 프로젝트는 GitHub Flow 전략을 사용한다.

- main: 최종 완성본 브랜치
- feature/*: 기능 개발 브랜치
- docs/*: 문서 작업 브랜치
- fix/*: 버그 수정 브랜치

작업은 Issue 생성 후 기능별 브랜치에서 진행하며, 작업 완료 후 Pull Request를 생성하여 팀장의 검토를 받은 뒤 main 브랜치에 병합한다.

## 협업 규칙

1. main 브랜치에 직접 push하지 않는다.
2. 기능별로 Issue를 생성한 뒤 작업한다.
3. 각 팀원은 자신이 맡은 기능을 feature 브랜치에서 구현한다.
4. 작업 완료 후 Pull Request를 생성한다.
5. 팀장은 Pull Request를 검토한 뒤 승인 또는 수정 요청을 한다.
6. 문제가 없으면 main 브랜치에 병합한다.
